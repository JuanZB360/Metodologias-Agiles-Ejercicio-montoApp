1.	Problemática
Hoy en día muchas personas realizan ventas, rifas, etc.… 
y muchas veces llevar un registro de cobro para cobros pendientes es tedioso o se realiza a papel y lápiz lo que lo hace enredado y poco eficiente si un cliente realiza abonos, o encontrar a una persona en especifico para verificar cuanto está debiendo aún, se hace complicado.

2.	¿Qué tipo de aplicación crearán?
Página web

3.	Solución Propuesta
Se plantea crea una aplicación que facilite el cobro de cualquier tipo de venta, la aplicación permitirá ingresar registros de ventas por producto y precio permitiendo ingresar si el pago esta completo, si pagara mas adelante y en qué fecha, nombre y teléfono de la persona a la que se le fiara el producto, además la aplicación deberá permitir ingresar abonos a las deudas.
4.	Usuarios del Sistema
Vendedor
5.	Funcionalidades Principales
-	Agregar producto
-	Modificar Producto
-	Mostrar Producto
-	Agregar Ventas
-	Modificar Ventas
-	Mostrar Ventas
-	Eliminar Ventas

6.	Flujo del Sistema 
-	Usuario se Registra en la aplicación -> Register.
-	Usuario ingresa a la aplicación -> login.
-	El usuario agrega Productos (Nombre, Precio, Cantidad, Descripción).
-	El usuario puede ver su inventario.
-	Usuario puede modificar Productos.
-	El usuario puede crear una venta (Producto, Cantidad, Precio, Modo de pago, fecha de pago, Monto abonado, Nombre cliente, Número de teléfono, Correo, Documento, Dirección).
-	 El usuario puede modificar una venta (Abono, fecha de pago).
